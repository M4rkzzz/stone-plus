## 版本亮点

### 🧠 Compact V2 按来源能力适配

- Codex 的 Compact V2 请求会优先路由到支持原生 opaque compact 的官方来源。
- 只有中转来源时，Stone+ 会在网关内部使用普通 Responses 生成摘要，再转换成客户端可继续处理的兼容终止帧。
- 不支持远程压缩的中转站不再直接触发 422；真正不完整的上游响应仍会按错误返回，不伪造成功。

### 🔁 失败切换与会话连续性

- compact 请求按实际凭据能力重新筛选账号，避免把 API Key、OAuth 和 Agent Identity 混在同一路径中。
- 原生 compact 无可用账号时可安全降级到兼容来源，并保留请求级失败切换与容量释放。
- 压缩摘要进入中转前会脱敏凭据内容，客户端会收到可继续对话的结构化 Responses 结果。

### 📡 流式推理进度更稳定

- 为 OpenAI、Anthropic 和 Gemini 的推理/思考增量增加无内容进度信号，避免长推理期间被误判为空闲。
- Gemini thinking 内容不会被错误地当作最终回答文本发送给仅需答案的兼容消费者。
- 普通文本、工具调用、终止事件和现有流式用量统计保持原有协议行为。

### 🛡️ 发布质量门修正

- Source Available 许可证边界校验改为“当前版本不得早于 0.9.6”，后续 0.9.x 版本可以正常发布。
- 继续保留身份、签名、许可证、运行时完整性和 provenance 发布门禁。

## 下载

- Windows x64 推荐下载 `StonePlus-0.9.7-windows-x64-setup.exe`；免安装使用 `StonePlus-0.9.7-windows-x64-portable.exe`。
- macOS 提供 Intel 与 Apple Silicon 的 `.dmg` 和 `.zip`；Linux 提供 x64/arm64 的 AppImage 与 deb。
- `StonePlus-0.9.7-source.tar.gz` 是与签名标签对应的源码快照；同时提供随包 sing-box 对应源码归档。
- `SHA256SUMS`、GitHub provenance 和 `StonePlus-CodeSigning.cer` 用于校验下载来源和完整性。

> Windows 使用带时间戳的项目自签名 Authenticode 证书，仍可能出现 SmartScreen 提示；macOS 为 ad-hoc 签名且未经过 Apple 公证；Linux 上安全保险库不可用时，涉及敏感凭据的备份会按安全策略阻止执行。
